## 1. 架构总览

```
用户请求 → FastAPI → [选择引擎] → 6-Agent 管线
                                  ├─ LangGraph（默认）: script→character→storyboard→image→voice→video
                                  └─ Runtime v2.0（USE_RUNTIME=true）: 同6步 + Hook质量门禁 + Memory人物一致性 + Skill驱动
                                         │
                                         ↓
                              PostgreSQL  Redis  LLM API  ComfyUI  CosyVoice  FFmpeg
```

**核心文件结构：**

```
storyflow-ai/
├── backend/
│   ├── main.py                    # FastAPI 入口
│   ├── .env                       # 环境变量配置（必填）
│   ├── configs/settings.py        # Pydantic Settings
│   ├── models/                    # SQLAlchemy 模型
│   ├── api/                       # API 路由 (story, task)
│   ├── agents/                    # 6个Agent
│   │   ├── script_agent.py        # 剧本Agent (LLM)
│   │   ├── character_agent.py     # 角色Agent (LLM)
│   │   ├── storyboard_agent.py    # 分镜Agent (LLM)
│   │   ├── image_agent.py         # 绘图Agent (ComfyUI/SD)
│   │   ├── voice_agent.py         # 语音Agent (CosyVoice/TTS)
│   │   └── video_agent.py         # 合成Agent (FFmpeg)
│   ├── workflows/
│   │   ├── story_workflow.py      # LangGraph 图构建
│   │   └── runtime_workflow.py    # Runtime 引擎入口
│   ├── runtime/                   # Agent OS Runtime v2.0
│   │   ├── adapter.py             # Agent适配器（Memory注入+质量门禁+Skill驱动）
│   │   ├── hook/                  # Hook框架（质量门禁）
│   │   ├── memory/                # Memory系统（人物一致性）
│   │   ├── conversation/          # 会话管理
│   │   └── skills/                # Skill定义目录 (YAML)
│   ├── skills/                    # 6个Agent的Skill定义 (YAML)
│   ├── tasks/runner.py            # 任务执行器（增量持久化）
│   ├── requirements.txt
│   └── Dockerfile
├── frontend/                      # React 18 + Ant Design 5 + Vite 5
│   ├── package.json
│   ├── vite.config.ts             # 代理 /api → localhost:8000
│   └── src/

│       └── api/index.ts           # API封装 + WebSocket
├── deploy/
│   ├── docker-compose.yml         # PostgreSQL + Redis + Qdrant + Backend + Nginx
│   └── nginx/default.conf
```

## 2. 两条引擎路径说明

| 特性 | LangGraph（默认） | Runtime v2.0 |
|------|-------------------|-------------|
| 触发方式 | `USE_RUNTIME=false`（默认） | `USE_RUNTIME=true` |
| 管线流程 | 6-Agent串行图 | 同6步 + Hook/Memory/Skill |
| 质量门禁 | 无（Agent出错直接失败） | 每步AFTER_AGENT校验，失败自动重试2次 |
| 人物一致性 | 无 | CharacterMemoryService跨集数保持外观一致 |
| Skill约束 | 无 | YAML定义I/O Schema、约束规则 |
| 可观测性 | 基础日志 | StructuredLogger + Langfuse |
| 稳定性 | 已验证 | 已验证（失败自动fallback到LangGraph） |

**建议**：先用 LangGraph 跑通基础流程，确认所有外部服务正常后，再开启 Runtime v2.0 享受增强能力。

## 3. 外部服务依赖清单

### 3.1 必须服务（缺一不可）

| 服务 | 用途 | 默认地址 | 安装方式 |
|------|------|---------|---------|
| **PostgreSQL 16** | 数据持久化 | `localhost:5432` | `docker run -d -p 5432:5432 -e POSTGRES_USER=storyflow -e POSTGRES_PASSWORD=storyflow -e POSTGRES_DB=storyflow postgres:16-alpine` |
| **Redis 7** | 任务状态缓存 + PubSub进度推送 | `localhost:6379` | `docker run -d -p 6379:6379 redis:7-alpine` |
| **LLM API** | 剧本/角色/分镜生成 | 取决于供应商 | 需要API Key |
| **ComfyUI** | SD图像生成 | `localhost:8188` | 需单独部署（见§5.4） |
| **CosyVoice** | TTS语音合成 | `localhost:50000` | 需单独部署（见§5.5） |
| **FFmpeg** | 视频合成 + 字幕烧录 | 系统命令 | `apt install ffmpeg` 或 Docker已内置 |

### 3.2 可选服务

| 服务 | 用途 | 缺少时的影响 |
|------|------|-------------|
| **Qdrant** | Runtime Memory 向量存储 | 主管线不依赖，仅 Runtime v2.0 Memory 子系统需要 |
| **Langfuse** | LLM调用追踪 | 完全可选，已内置优雅降级 |

### 3.3 LLM API 兼容性

项目使用 `langchain-openai.ChatOpenAI`，兼容所有 OpenAI API 格式的服务：

| 供应商 | BASE_URL | MODEL | 说明 |
|--------|----------|-------|------|
| OpenAI | `https://api.openai.com/v1` | `gpt-4o` | 原版 |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` | 性价比高，推荐测试用 |
| 智谱GLM | `https://open.bigmodel.cn/api/paas/v4` | `glm-4-flash` | 国内直连 |
| Moonshot | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` | 国内直连 |
| 本地Ollama | `http://localhost:11434/v1` | `qwen2.5:14b` | 零成本，需要GPU |

## 4. 方式一：Docker Compose 一键部署（推荐）

适用于快速验证，PostgreSQL + Redis + Qdrant 一键启动。

### 4.1 前提条件

- 已安装 Docker + Docker Compose
- 已部署 ComfyUI 和 CosyVoice（见§5.4、§5.5）
- 有 LLM API Key

### 4.2 启动基础设施

```bash
cd storyflow-ai/deploy

# 创建 .env（在deploy目录下）
cp ../backend/.env .env
# 编辑 .env，填入 LLM_API_KEY

# 启动 PostgreSQL + Redis + Qdrant
docker compose up -d postgres redis qdrant

# 等待健康检查通过
docker compose ps
```

### 4.3 构建并启动后端

```bash
# 构建后端镜像
docker compose build backend

# 启动后端
docker compose up -d backend

# 查看日志
docker compose logs -f backend
```

### 4.4 启动前端（开发模式）

```bash
cd ../frontend
npm install
npm run dev
# 访问 http://localhost:3000
```

### 4.5 注意事项

- Docker Compose 中 ComfyUI/CosyVoice 使用 `host.docker.internal` 访问宿主机服务
- **Linux** 上 `host.docker.internal` 默认不可用，需在 `docker compose up` 时加 `--add-host=host.docker.internal:host-gateway`
- 生产环境建议用 Nginx 反向代理（docker-compose 中已配置）

## 5. 方式二：手动本地部署

### 5.1 系统要求

```
Python 3.11+
Node.js 18+
FFmpeg (系统级)
```

### 5.2 启动基础设施

```bash
# PostgreSQL
docker run -d \
  --name storyflow-postgres \
  -p 5432:5432 \
  -e POSTGRES_USER=storyflow \
  -e POSTGRES_PASSWORD=storyflow \
  -e POSTGRES_DB=storyflow \
  -v storyflow-pg:/var/lib/postgresql/data \
  postgres:16-alpine

# Redis
docker run -d \
  --name storyflow-redis \
  -p 6379:6379 \
  redis:7-alpine

# Qdrant（可选，仅 Runtime v2.0 Memory 需要）
docker run -d \
  --name storyflow-qdrant \
  -p 6333:6333 \
  -v storyflow-qdrant:/qdrant/storage \
  qdrant/qdrant:latest

# FFmpeg
# Ubuntu/Debian: sudo apt install ffmpeg
# macOS: brew install ffmpeg
# 验证: ffmpeg -version
```

### 5.3 配置后端

```bash
cd storyflow-ai/backend

# 1. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
cp .env .env.backup   # 备份一下
# 编辑 .env，至少修改以下项：
```

**必须修改的 `.env` 项：**

```bash
# LLM（必填）
LLM_API_KEY=sk-your-real-api-key-here
LLM_MODEL=gpt-4o
LLM_BASE_URL=https://api.openai.com/v1

# 如果用 DeepSeek 测试：
# LLM_API_KEY=sk-your-deepseek-key
# LLM_MODEL=deepseek-chat
# LLM_BASE_URL=https://api.deepseek.com/v1
```

**如果 ComfyUI / CosyVoice 不在本机，也需修改：**

```bash
COMFYUI_URL=http://你的comfyui地址:8188
COSYVOICE_URL=http://你的cosyvoice地址:50000
```

### 5.4 部署 ComfyUI（图像生成）

ComfyUI 需要独立部署，用于 SDXL 图像生成。

**快速启动：**

```bash
# 克隆 ComfyUI
git clone https://github.com/comfyanonymous/ComfyUI.git
cd ComfyUI

# 安装依赖
pip install -r requirements.txt

# 下载模型（SDXL 推荐）
# 将模型文件放到: ComfyUI/models/checkpoints/
# 推荐模型:
#   - stable-diffusion-xl-base-1.0.safetensors
#   - revAnimated_v2.safetensors （image_agent 默认使用）

# 启动
python main.py --listen 0.0.0.0 --port 8188
```

**验证：** 浏览器访问 `http://localhost:8188` 能看到 ComfyUI 界面。

**注意：** `image_agent.py` 默认引用 `stable-diffusion/revAnimated_v2.safetensors`，请确保该模型存在于 ComfyUI 的 `models/checkpoints/` 目录下，或在代码中修改模型路径。

### 5.5 部署 CosyVoice（语音合成）

CosyVoice 需要独立部署，用于 TTS 语音合成。

**快速启动：**

```bash
# 克隆 CosyVoice

cd CosyVoice

# 安装依赖

# 下载模型（按 CosyVoice 官方文档）

#   POST /voice/generate
#   Body: { "text": "台词内容", "speaker": "male_1" 或 "female_1", "speed": 1.0 }
#   Response: { "audio": "<base64编码的音频>" }  或  { "audio_url": "http://..." }
```

**验证：**

```bash
curl -X POST http://localhost:50000/voice/generate \
  -H "Content-Type: application/json" \
  -d '{"text": "你好世界", "speaker": "female_1"}'
```

## 6. 配置详解

### 6.1 完整 `.env` 模板

```bash
# ===== 应用 =====
DEBUG=true                    # 开发模式（CORS允许所有来源）

# ===== LLM（必填） =====
LLM_API_KEY=sk-xxx            # 你的 API Key
LLM_MODEL=gpt-4o              # 模型名
LLM_BASE_URL=https://api.openai.com/v1  # API 地址
# LLM_TEMPERATURE=0.7          # 默认温度（各Agent会覆盖）
# LLM_MAX_TOKENS=4096          # 默认最大token

# ===== 数据库 =====
DATABASE_URL=postgresql+asyncpg://storyflow:storyflow@localhost:5432/storyflow

# ===== Redis =====
REDIS_URL=redis://localhost:6379/0

# ===== Qdrant（可选） =====
QDRANT_URL=http://localhost:6333

# ===== ComfyUI =====
COMFYUI_URL=http://localhost:8188
# COMFYUI_POLL_TIMEOUT=300     # 单图等待超时（秒）
# COMFYUI_MAX_RETRIES=2        # 单图重试次数

# ===== CosyVoice =====
COSYVOICE_URL=http://localhost:50000

# ===== 存储 =====
STORAGE_PATH=./storage         # 生成物存放路径

# ===== Runtime v2.0 =====
USE_RUNTIME=false              # true=Runtime引擎（含质量门禁+人物一致性+Skill驱动）

# ===== Langfuse（可选） =====
# LANGFUSE_PUBLIC_KEY=
# LANGFUSE_SECRET_KEY=
# LANGFUSE_HOST=https://cloud.langfuse.com

# ===== 生成参数 =====
# MAX_EPISODES=6               # 最大集数
# SCENES_PER_EPISODE=[5,10]    # 每集场景数范围

# ===== Memory =====
# MEMORY_SESSION_TTL=86400     # 记忆会话TTL
# MEMORY_CONFIDENCE_THRESHOLD=0.7  # 一致性校验阈值
```

### 6.2 各Agent的LLM参数覆盖

| Agent | Temperature | Max Tokens | 说明 |
|-------|------------|------------|------|
| script_agent | 0.8 | 8192 | 创意性高 |
| character_agent | 0.4 | 4096 | 精确描述 |
| storyboard_agent | 0.4 | 4096 | 精确分镜 |

## 8. 运行第一个漫剧生成

### 8.1 通过前端

1. 打开 `http://localhost:3000`
2. 填写：
   - **标题**: 测试漫剧
   - **类型**: 奇幻
   - **提示词**: 一个勇敢的少年在魔法森林中寻找失落宝剑的故事
3. 点击「开始创作」
4. 等待进度走完（6个步骤：剧本→角色→分镜→绘图→配音→合成）
5. 完成后点击「查看结果」观看视频

### 8.2 通过API

```bash
# 1. 创建故事
curl -X POST http://localhost:8000/api/story \
  -H "Content-Type: application/json" \
  -d '{
    "title": "测试漫剧",
    "genre": "奇幻",
    "prompt": "一个勇敢的少年在魔法森林中寻找失落宝剑的故事"
  }'
# 返回: {"id": "xxx", "title": "测试漫剧", ...}

# 2. 启动生成（用上一步返回的id）
curl -X POST http://localhost:8000/api/story/{story_id}/generate
# 返回: {"task_id": "xxx", "message": "Generation started"}

# 3. 查询进度
curl http://localhost:8000/api/task/{task_id}
# 返回: {"status":"processing","progress":50,"current_step":"image",...}

# 4. WebSocket 实时进度
# ws://localhost:8000/api/task/{task_id}/ws

# 5. 获取结果（完成后）
curl http://localhost:8000/api/story/{story_id}/result
```

### 8.3 生成物位置

```

    └── {story_id}/
        ├── images/          # 每个场景的SD生成图片
        │   ├── scene_001.png
```

## 9. 切换引擎模式

### 9.1 LangGraph 模式（默认）

```bash
# .env
USE_RUNTIME=false
```

- 直接走 LangGraph 6-Agent 串行图
- 简单可靠，无额外开销
- 出错直接失败，无自动重试

### 9.2 Runtime v2.0 模式

```bash
# .env
USE_RUNTIME=true
```

- 启用 Hook 质量门禁：每个Agent执行后自动校验输出，不合格重试最多2次
- 启用 Memory 人物一致性：character_agent 产出角色外观 → 自动存储 → storyboard/image Agent 自动注入一致性约束
- 启用 Skill 驱动：每个Agent有 YAML 定义的约束规则，生成时自动校验
- **失败自动 fallback** 到 LangGraph 模式，不会卡住

### 9.3 Runtime v2.0 质量门禁规则

| Agent | 校验内容 |
|-------|---------|
| script | 大纲长度 > 100字、角色数 2-8、每集内容完整 |
| character | 四维度外观(hair/body/cloth/face)齐全 |
| storyboard | 场景数合理、角色引用完整、prompt长度、时长3-15秒 |
| image | 产出图片文件存在 |
| voice | 产出音频文件存在 |
| video | 产出视频文件存在 |

## 10. 常见问题排查

### Q1: 启动报 "Database init failed"

```
原因: PostgreSQL 未启动 或 连接信息错误
排查:
  1. docker ps 确认 postgres 容器在运行
  2. 检查 .env 中 DATABASE_URL 是否正确
  3. 手动连接测试: psql -h localhost -U storyflow -d storyflow
```

### Q2: 启动报 "Redis not available"

```
原因: Redis 未启动
排查:
  1. docker ps 确认 redis 容器在运行
  2. 检查 .env 中 REDIS_URL 是否正确
  3. 手动连接测试: redis-cli ping
```

### Q3: 创建故事后启动生成，一直卡在 "script" 步骤

```
原因: LLM API 不可用 或 API Key 错误
排查:
  1. 检查 .env 中 LLM_API_KEY 是否填了真实 Key
  2. 测试 API 连通性:
     curl https://api.openai.com/v1/chat/completions \
       -H "Authorization: Bearer sk-xxx" \
       -H "Content-Type: application/json" \
       -d '{"model":"gpt-4o","messages":[{"role":"user","content":"hi"}]}'
  3. 检查 LLM_BASE_URL 是否正确（国内代理注意URL）
  4. 查看后端日志中的 tenacity retry 信息
```

### Q4: image 步骤全部失败

```
原因: ComfyUI 未启动 或 模型文件缺失
排查:
  1. 浏览器访问 http://localhost:8188 确认 ComfyUI 界面可用
  2. 检查 ComfyUI models/checkpoints/ 下是否有模型文件
  3. 确认 image_agent.py 中引用的模型名与文件名一致
  4. ComfyUI GPU 显存不足时也会失败，检查 GPU 使用率
```

### Q5: voice 步骤失败

```
原因: CosyVoice 未启动 或 API 接口不匹配
排查:
  1. curl 测试 CosyVoice /voice/generate 接口
  2. 确认返回格式包含 "audio" (base64) 或 "audio_url" 字段
  3. 如果所有场景无对话，voice 步骤会跳过（这是正常的）
```

### Q6: video 步骤失败

```
原因: FFmpeg 未安装 或 图片/音频路径问题
排查:
  1. 终端运行 ffmpeg -version 确认已安装
  2. 检查 storage/stories/{id}/images/ 下是否有图片
  3. video_agent 有多层 fallback（缺图片跳过、缺音频生成静音），通常不会完全失败
```

### Q7: 前端页面刷新后进度丢失

```
已修复: 现在刷新页面会自动通过 /api/task/by-story/{story_id} 恢复 task_id，
重新连接 WebSocket 继续接收进度。
```

### Q8: Runtime 模式下启动失败

```
排查:
  1. 检查日志中 Runtime init 是否报错
  2. Runtime 初始化失败不影响 LangGraph 路径，会自动 fallback
  3. 如果 skills/ 目录不存在，Runtime 仍可运行（Skill 为空壳，不影响核心流程）
```

### Q9: Docker Linux 上无法访问宿主机 ComfyUI

```
原因: Linux 上 host.docker.internal 默认不可用
解决:
  1. docker compose up 时加 --add-host=host.docker.internal:host-gateway
  2. 或者修改 docker-compose.yml，在 backend 服务中加:
     extra_hosts:
       - "host.docker.internal:host-gateway"
  3. 或者直接用宿主机内网 IP 替换 host.docker.internal
```

### Q10: 想零成本跑通测试（不用 LLM / ComfyUI / CosyVoice）

```
当前项目没有内置 mock 模式，至少需要 LLM API 才能跑通前3步。

最小成本方案:
  1. LLM: 用 Ollama 本地部署 qwen2.5:14b（零API费用，需要GPU）
     LLM_BASE_URL=http://localhost:11434/v1
     LLM_MODEL=qwen2.5:14b
     LLM_API_KEY=ollama
  2. ComfyUI: 本地部署（需要GPU，显存≥8GB）
  3. CosyVoice: 本地部署（需要GPU）
  4. 如果只想测试前3步（script→character→storyboard）:
     可以不启动 ComfyUI/CosyVoice，但需要在代码中
     让 image/voice/video agent 返回 mock 数据
```
