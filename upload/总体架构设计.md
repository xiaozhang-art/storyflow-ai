# 📘 00 - StoryFlow Runtime 总体架构设计（V1）

---

# 1. 项目定位

StoryFlow Runtime 不是一个应用，而是一个：

> **Multi-Agent Runtime Framework（多智能体运行时框架）**

用于支持：

* Agent 间通信（A2A）
* Skill 调度
* Tool 执行
* Memory 管理
* Workflow 编排
* Hook 观测
* Evaluation 反馈闭环

# 3. 核心设计原则

## 3.1 Agent 无状态

Agent 不保存 session、不保存通信关系。

👉 所有状态由 Runtime 管理

---

## 3.2 所有通信通过 Message Bus

禁止：

```python
script.run(storyboard)
```

必须：

```python
send_message({
  from: "script",
  to: "storyboard",
  action: "generate"
})
```

---

## 3.3 Skill > Agent

Agent = 调度器
Skill = 真正能力单元

---

## 3.4 Hook 全面接管可观测性

所有执行必须经过 Hook：

* LLM
* Tool
* Skill
* Memory
* Retry

---

## 3.5 Runtime 是唯一真状态中心

所有状态：

* session
* conversation
* trace
* memory
* artifacts

全部在 Runtime

---

# 4. 核心模块

---

# 4.1 Agent Runtime

职责：

* 执行 Agent
* 注入 Skill
* 注入 Memory
* 调用 LLM
* 触发 Hook

接口：

```python
class AgentRuntime:

    async def run(self, envelope: Envelope) -> Envelope:
        pass
```

内部流程：

```text
receive envelope
  ↓
load memory
  ↓
load skills
  ↓
hook BEFORE_AGENT
  ↓
select skill
  ↓
call LLM
  ↓
call tool (optional)
  ↓
judge (optional)
  ↓
hook AFTER_AGENT
  ↓
return envelope
```

---

# 4.2 Message Bus（A2A）

核心数据结构：

```python
class Envelope:
    trace_id: str
    conversation_id: str
    message_id: str
    parent_id: str

    sender: str
    receiver: str

    action: str
    payload: dict

    artifacts: list
    memory_refs: list
    metadata: dict
```

---

路由逻辑：

```text
Envelope
  ↓
Router
  ↓
Agent Registry
  ↓
Target Agent Runtime
```

---

# 4.3 Conversation Manager

职责：

* 管理 agent group
* 管理 message flow
* 管理 DAG

结构：

```python
Conversation:
    id
    participants
    state
```

---

# 4.4 Session Manager

⚠️ Session 只属于 Runtime，不暴露给 Agent

```python
Session:
    session_id
    conversation_id
    agent_id
    context_window
```
